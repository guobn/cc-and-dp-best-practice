# DeepMind — 多风格 AI 知识工作台

> 在同一页面中集成 6 种 Web 设计风格，点击即可切换，切换时有明显转场效果。纯前端 Demo，无后端依赖。

---

## 项目定位

这是一个**风格实验性质**的 AI 知识工作台 Demo。它在一个页面中集成了四种功能模块（智能采集、知识对话、工作流编排、成果产出），但核心亮点不在于功能，而在于**视觉风格的多样性**和**风格切换的转场体验**。

你可以把它看作一个"设计风格主题公园"—— 用户可以在六种截然不同的视觉世界间来回穿梭，而业务数据始终保持完整。

## 6 种风格介绍

### 1. Glassmorphism（玻璃拟态）

半透明毛玻璃卡片悬浮在渐变背景之上，模糊与光影交织，营造出轻盈、现代的氛围。

**关键 Design Tokens：**

| Token | 值 |
|-------|-----|
| 背景 | `linear-gradient(135deg, #667eea, #764ba2, #f093fb)` |
| 卡片背景 | `rgba(255,255,255,0.12)` + `backdrop-filter: blur(16px)` |
| 卡片边框 | `1px solid rgba(255,255,255,0.2)` |
| 卡片圆角 | `16px` |
| 卡片阴影 | `0 8px 32px rgba(0,0,0,0.1)` |
| 字体颜色 | `rgba(255,255,255,0.9)` |
| 字体 | System UI Sans |
| 交互反馈 | hover 时背景不透明度提升，边框光晕增强 |

---

### 2. Neo-Brutalism（新粗野主义）

粗犷、大胆、不修边幅。厚实的黑色边框、错位的硬阴影、高饱和色块、全大写的无衬线标题，带着一种"我就这样，你爱咋咋地"的态度。

**关键 Design Tokens：**

| Token | 值 |
|-------|-----|
| 背景 | `#fffbe6`（暖黄奶油色） |
| 卡片边框 | `4px solid #000` |
| 卡片阴影 | `8px 8px 0 #000`（错位硬阴影） |
| 卡片圆角 | `0`（直角） |
| 标题字体 | `Impact / Arial Narrow Bold`（粗壮无衬线） |
| 标题装饰 | 全大写 + 左边框 |
| 色块点缀 | `#ffd700`（金）、`#ff6b6b`（红）、`#51cf66`（绿） |
| 按钮阴影 | `5px 5px 0 #000`，按下时位移 |

---

### 3. Neumorphism（新拟物 / Soft UI）

单色背景上，元素通过双向软阴影（外凸 + 内凹）模拟出从背景中"挤压"出来的立体感，像物理世界中的浮雕。

**关键 Design Tokens：**

| Token | 值 |
|-------|-----|
| 背景 | `#e0e5ec`（统一中性灰） |
| 卡片阴影（外凸） | `9px 9px 16px #b8bcc3`（暗影） |
| 卡片阴影（高光） | `-9px -9px 16px #ffffff`（亮影） |
| 卡片圆角 | `15px` |
| 内凹效果 | `inset` 阴影用于输入框和激活的标签页 |
| 字体颜色 | `#333` / `#555` |
| 激活反馈 | 外凸 → 内凹（按下感） |

---

### 4. Cyberpunk / Retrofuturism（赛博朋克）

黑色背景上，霓虹青和品红交织。故障特效、扫描线、CRT 闪烁、等宽字体 —— 仿佛走进了《银翼杀手》的世界。

**关键 Design Tokens：**

| Token | 值 |
|-------|-----|
| 背景 | `#0a0a0a`（纯黑） |
| 主色（青） | `#00fff9` 霓虹青 |
| 辅色（品红） | `#ff00ff` 霓虹品红 |
| 卡片边框 | `1px solid rgba(0,255,249,0.3)` |
| 卡片发光 | `0 0 10px rgba(0,255,249,0.1)` |
| 标题字体 | `Orbitron`（等宽未来感） |
| 正文字体 | `Share Tech Mono`（等宽） |
| 特效 | 扫描线覆盖层、CRT 闪烁、Glitch 故障动画 |
| 按钮 | 透明背景 + 霓虹边框 + 发光 hover |

---

### 5. Minimalism（极简主义）

极致留白、单一中性色、几乎无装饰的细体字。每一个元素的存在都必须有充分的理由。

**关键 Design Tokens：**

| Token | 值 |
|-------|-----|
| 背景 | `#ffffff` 纯白 |
| 文字颜色 | `#333333` |
| 次要文字 | `#999999` / `#cccccc` |
| 卡片背景 | `#fafafa`（极淡灰） |
| 卡片边框 | 无 |
| 卡片圆角 | 0（强制直角） |
| 字体粗细 | 200-300（极致轻盈） |
| 字间距 | 1-3px（宽松） |
| 装饰 | 仅极淡的 `1px solid #eee` 分割线 |
| 按钮 | 纤细边框 + 透明背景，hover 时仅加深边框 |

---

### 6. Maximalism / Y2K（千禧风）

高饱和对比色、多种字体混搭、重叠图形、贴纸装饰、轻微旋转 —— 它是极简主义的反面，热烈、张扬、充满个性。

**关键 Design Tokens：**

| Token | 值 |
|-------|-----|
| 背景 | `linear-gradient(135deg, #ffecd2, #fcb69f, #ff9a9e)` |
| 主色 | `#ff6b6b`（红）、`#ffd93d`（黄）、`#6bcb77`（绿）、`#4d96ff`（蓝） |
| 卡片边框 | `3px solid`（颜色随卡片变化） |
| 卡片阴影 | `5px 5px 0`（颜色随边框变化） |
| 卡片旋转 | `0.3deg ~ 0.8deg` 轻微随机旋转 |
| 标题字体 | `Fredoka One`（圆润可爱） |
| 正文字体 | `Comic Neue`（手写风格） |
| 装饰 | emoji 贴纸、渐变文字、彩虹分割线、浮动装饰 |
| 字体数量 | 3 种混搭 |

## 切换转场原理

风格切换的核心机制是 `style-switcher.js`，其工作流程如下：

### 1. 状态管理

- 每个风格对应一个 CSS class 前缀（`.style-glass`、`.style-brutal` 等）
- 所有风格 CSS 使用 class 前缀限定作用域，彼此完全隔离
- 当前风格名存储在 `AppState.currentStyle` 中
- 切换时仅修改 `<body>` 的 class，业务数据（知识库、聊天记录等）不受影响

### 2. 三种转场方案（随机选用）

每次切换时随机选择一种方案：

#### 方案 A：全屏蒙版扫过（Sweep）

```
时间线：
1. [0ms]     创建一个全屏 overlay，初始 scaleX=0（不可见）
2. [0-400ms] overlay 从左侧向右扫过，scaleX 从 0 → 1（完全覆盖屏幕）
3. [400ms]   在完全覆盖的瞬间，切换 body class（旧风格 → 新风格）
4. [400-800ms] overlay 从右侧向左扫出，scaleX 从 1 → 0（新风格揭晓）
```

使用 GSAP 的 `power2.inOut` 缓动函数，产生平滑的加速减速效果。overlay 的颜色与目标风格匹配（如赛博朋克用霓虹渐变，粗野主义用金黄）。

#### 方案 B：碎片化解构重组（Fragment）

```
时间线：
1. [0ms]     创建 8x8 碎片网格（64 个色块），初始分散在页面各处
2. [0-500ms] 碎片飞向中心聚合，覆盖屏幕
3. [500ms]   碎片完全聚合时，切换 body class
4. [500-1000ms] 碎片飞散（带随机偏移和旋转），露出新风格
5. [1000ms]  移除碎片网格
```

每个碎片使用 `cubic-bezier(0.68, -0.55, 0.27, 1.55)` 缓动曲线，产生弹性的"回弹"效果。

#### 方案 C：液态形变（Morph）

```
时间线：
1. [0ms]     overlay 的 clip-path 设置为 circle(0%)
2. [0-500ms] clip-path 动画到 circle(70.7%)，使用 elastic.inOut 缓动
3. [500ms]   完全覆盖时切换 body class
4. [500-1000ms] clip-path 动画回 circle(0%)，揭晓新风格
```

### 3. 状态保持

风格切换**只修改 body 的 class**，不销毁或重建任何 DOM 元素。因此：
- 知识库内容不变
- 对话历史保留
- 流水线状态保持（如果正在运行，切换后继续运行）
- 滚动位置不变
- 输入框内容不变

## 如何新增第 7 种风格（分步指南）

### 步骤 1：在 `styles/` 下创建新 CSS 文件

```css
/* styles/retro.css */
.style-retro body {
  background: #f4e4c1;
  font-family: 'Times New Roman', serif;
  /* ... 其他样式覆盖 */
}
```

### 步骤 2：在 index.html 中引入

```html
<link rel="stylesheet" href="styles/retro.css">
```

### 步骤 3：在 style-switcher.js 中注册

在 `STYLE_META` 对象中添加新条目：

```js
const STYLE_META = {
  // 其他风格...
  retro: { name: '复古风', emoji: '\uD83D\uDCFB', icon: '\uD83D\uDCFB' }
};
```

### 步骤 4：在 style-switcher.js 的 applyStyle 函数中

确保 `Object.keys(STYLE_META)` 能动态获取所有风格名 — 这段代码已经是动态的，所以不需要修改。

### 步骤 5：在 app.js 的 AppState 中（可选）

```js
const AppState = {
  // ...
  currentStyle: 'glass' // 可改为 'retro' 作为默认
};
```

### 步骤 6：编写 CSS 覆盖规则

建议覆盖以下核心元素（共约 20-30 条规则）：

```
body, #app, .app-header, .logo,
.style-btn,
.ai-sidebar, .ai-sidebar-item,
.module-tabs, .tab-btn,
.card, .knowledge-card, .tag,
.btn, .input-field,
.chat-message, .chat-message .citation-link,
.pipeline-node, .pipeline-connector, .pipeline-output,
.deliverable-card, .deliverable-preview,
.toast,
h1, h2, h3, p
```

## 设计取舍

### 1. 单一 HTML 文件 vs. 组件化

取舍：本项目使用单一 HTML 文件包含所有模块的 DOM 结构。

权衡：组件化架构（如 React/Vue）更适合大型项目，但本项目核心目标是展示视觉风格而非组件抽象。单一 HTML 结构更简单、加载更快，也便于理解风格切换的核心机制。

### 2. 6 个独立 CSS 文件 vs. 单一 CSS 变量主题

取舍：每个风格使用独立的 CSS 文件，而非通过 CSS 变量切换主题。

权衡：CSS 变量更适合"换肤"场景（改几个颜色值），但本项目要求的不只是换色 — 每种风格需要改变布局、字体、阴影、圆角、动效曲线甚至元素形状。独立 CSS 文件提供了完全的自由度，适合这种"彻底颠覆"的需求。代价是存在少量样式冗余。

### 3. 类前缀隔离 vs. Shadow DOM

取舍：使用 `.style-glass .card` 类前缀实现样式隔离。

权衡：Shadow DOM 提供了更严格的隔离，但会使 GSAP 动画和全局事件处理变得更复杂。类前缀方案虽然不够"干净"，但对于 Demo 项目来说更简单、更透明。

### 4. GSAP vs. CSS Animation

取舍：转场动画同时使用 GSAP 和 CSS Animation，GSAP 为主，CSS 为 fallback。

权衡：GSAP 提供了更精细的动画控制（时间线、缓动函数、回调），特别适合"蒙版扫过"这类需要精确时间点的动画。CSS Animation 作为 fallback，确保在没有 GSAP CDN 时仍有基本转场效果。

### 5. 预置模拟数据 vs. 真实 API

取舍：所有 AI 行为使用预置数据 + 动画模拟。

权衡：这是故意为之的设计决策 — 项目的核心价值在于视觉展示，而非后端功能。模拟数据让项目零依赖即可运行，同时通过动画延迟和状态机模拟，给用户接近真实的交互体验。

### 6. 毛玻璃的性能成本

在 Glassmorphism 风格中，大量使用 `backdrop-filter: blur()` 对性能有较大影响。在低端设备或 Safari 浏览器上可能出现卡顿。这是一个已知的权衡 — 视觉效果的提升以性能为代价。

### 7. 字体加载延迟

Cyberpunk 和 Maximal 风格依赖 Google Fonts 加载特殊字体。在字体加载完成前，页面会使用回退字体显示，可能导致短暂的"闪烁"。可以通过 `font-display: swap` 缓解，但无法完全消除。

## 构建日志

### 2026-05-18 构建记录

说实话，接到这个需求的时候我是有点懵的。六个风格、四种功能模块、三种转场动画、还要在一个纯前端的 HTML 文件里全部搞定——这听起来就像是那种"三天做完一个月的活"的需求。

我一开始的策略是先搭骨架：把四个功能模块的 HTML 结构写出来，不管样式。然后我发现光这个骨架就已经有将近 400 行了。好吧，没关系，毕竟单页应用嘛。

然后是数据层。我写了四个数据文件，每个模块一份。知识库我放了六张卡片，对话我写了六种不同关键词的回复模板（从 transformer 到多模态，覆盖了主流 AI 话题），流水线我设计了四个节点各带不同的执行时长和输出内容，产出物我搞了三份不同的 Markdown 文档。

最让我头疼的是样式。六个 CSS 文件，每个都要覆盖大约 20-30 个选择器。而且不能只是改颜色——Glassmorphism 要半透明毛玻璃，Brutalism 要厚边框错位阴影，Neumorphism 要双向软阴影，Cyberpunk 要霓虹光效加扫描线，Minimalism 要做到"仿佛什么都没做但就是好看"，Maximalism 要色彩炸裂还带 emoji 装饰...

写到第四个（Cyberpunk）的时候我已经在怀疑人生了——`box-shadow` 写了太多遍以至于形成肌肉记忆。写到第六个（Maximal）的时候反而挺开心的，因为终于可以放飞自我了：三种字体混搭、彩虹渐变、贴纸、旋转——怎么开心怎么来。

转场动画算是最有意思的部分。我实现了三种方案：蒙版扫过、碎片化解构、液态形变。本来只想做一个蒙版扫过就交差，但写完之后觉得不够过瘾，又加了碎片和液态。GSAP 的 `elastic.inOut` 缓动效果真的很棒，看着 clip-path 动画那个弹性回弹效果，我盯着屏幕看了好几遍。

最大的坑是：我最初想用 CSS 变量来实现主题切换，写到一半发现根本行不通——因为变量的值只能是颜色和长度，无法控制"这个风格用圆角那个风格用直角"这种结构性差异。最后全改成了类前缀方案，虽然代码量翻倍，但每个风格的表达力完全释放了。

验收阶段我列了一张自检清单，一条一条过。六个风格的 CSS 规则加起来接近 800 行，说真的到最后眼睛都快花了。但看到每一风格在屏幕上呈现出各自独特的气质时，还是有成就感的。

最惊喜的时刻是：在 cyberpunk 风格下跑流水线，四个节点的霓虹边框依次亮起，扫描线缓缓划过屏幕，配合 GSAP 的弹性动画——真的有了那么一点科幻电影的味道。

总的来说，这是一个"螺蛳壳里做道场"的项目。在纯前端的限制下，通过精心设计的 CSS 和动画，让一个页面拥有了六个完全不同的灵魂。
